# File Management in Linux

Linux file management commands are used to create, access, copy, move, delete, and manage files and directories.

These commands are commonly used by DevOps and SRE engineers during server management, troubleshooting, deployments, and log analysis.

# Common File Management Commands

**ls**
Lists files and directories.
ls

**ls -l**
Displays detailed file information.
ls -l

**pwd**
Displays current working directory.
pwd

**cd**
Changes directory.
cd directory_name

**mkdir**
Creates a new directory.
mkdir test-folder

**touch**
Creates a new empty file.
touch demo.txt

**cp**
Copies files or directories.
cp file1.txt file2.txt

**mv**
Moves or renames files.
mv file1.txt newfile.txt

**rm**
Removes files or directories.
rm file.txt

**find**
Searches for files and directories.
find . -name file.txt

# Why File Management is Important for DevOps/SRE

- Managing application files
- Checking logs
- Deployment activities
- Organizing scripts
- Troubleshooting server issues
- Handling configuration files
